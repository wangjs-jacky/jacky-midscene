# 作业任务目录

本目录用于记录学习过程中的所有作业任务和练习案例。

## 目录结构

建议按以下方式组织文件：

```
docs/plan/
├── stage1-1.1-环境搭建.md
├── stage1-1.2-playwright基础.md
├── stage1-1.3-桥接模式.md
├── stage1-1.4-yaml脚本.md
├── stage1-1.5-测试集成.md
└── ...
```

## 文件命名规范

- 按阶段和章节命名：`stage1-1.1-环境搭建.md`
- 按作业类型命名：`作业-商品搜索提取.md`
- 使用中文或英文都可以，保持一致性即可

## 文件模板

每个作业文件应包含以下内容：

```markdown
# 作业标题

## 作业描述
[作业的详细描述]

## 完成步骤
1. [步骤1]
2. [步骤2]
...

## 完成状态
- [ ] 未开始
- [ ] 进行中
- [x] 已完成

## 完成时间
[完成日期]

## 遇到的问题
[记录遇到的问题]

## 解决方案
[问题的解决方案]

## 学习总结
[完成作业后的总结和收获]
```

## 使用说明

1. 开始新作业时，创建对应的 Markdown 文件
2. 按照模板填写作业信息
3. 完成过程中及时更新状态
4. 遇到问题记录在"遇到的问题"部分
5. 完成后填写"学习总结"
