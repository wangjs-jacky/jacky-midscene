# 作业：YAML 脚本使用

## 作业描述
学习 YAML 脚本格式，理解 YAML 配置结构，掌握如何使用 YAML 脚本完成自动化任务。

## 学习目标
- 学习 YAML 脚本格式（`yaml-scripts-demo/`）
- 理解 YAML 配置结构：`web` 配置、`tasks` 任务列表、`flow` 步骤定义
- 运行 YAML 脚本示例

## 完成步骤

### 基础作业
1. [ ] 查看 `yaml-scripts-demo/midscene-scripts/` 中的所有示例脚本
2. [ ] 理解每个示例的配置结构和执行流程
3. [ ] 运行至少 3 个不同的示例脚本
4. [ ] 观察生成的报告和输出文件

### 实战作业 1 - 完整用户流程
1. [ ] 编写一个 YAML 脚本，自动化电商网站的购物流程：
   - 访问网站 → 搜索商品 → 查看商品详情 → 添加到购物车
2. [ ] 使用 `flow` 定义每个步骤
3. [ ] 使用 `assert` 验证每个步骤是否成功
4. [ ] 生成执行报告

### 实战作业 2 - 数据提取任务
1. [ ] 编写一个 YAML 脚本，从新闻网站提取数据：
   - 访问新闻网站首页
   - 提取所有新闻标题、链接、发布时间
   - 将数据保存为 JSON 文件
2. [ ] 使用 `output` 配置输出路径
3. [ ] 验证提取的数据格式是否正确

### 实战作业 3 - 多任务脚本
1. [ ] 创建一个包含多个 `tasks` 的 YAML 脚本
2. [ ] 每个任务执行不同的操作
3. [ ] 配置任务之间的依赖关系（如果有）
4. [ ] 运行脚本，观察任务执行顺序

## 思考问题
1. YAML 脚本和 TypeScript 脚本各自的优缺点是什么？什么时候应该用哪个？
2. YAML 脚本中的 `web` 配置项有哪些？它们的作用分别是什么？
3. `flow` 中的步骤是如何执行的？是顺序执行还是可以并行？
4. 如何在 YAML 脚本中处理错误？如何配置重试机制？
5. `output` 配置的作用是什么？可以输出哪些类型的数据？
6. YAML 脚本如何与桥接模式结合使用？
7. 如何调试 YAML 脚本？如果执行失败，如何定位问题？
