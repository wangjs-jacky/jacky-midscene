# 常见问题 - 环境配置

## Q1: 如何配置 LLM API Key？
**问题描述**：不知道在哪里配置 LLM API Key，配置后测试失败。

**答案**：
1. 在 `midscene-example/connectivity-test/` 目录下创建 `.env` 文件
2. 在 `.env` 文件中添加：
   ```
   OPENAI_API_KEY="your-api-key-here"
   ```
3. 确保 `.env` 文件在 `.gitignore` 中，不要提交到 Git
4. 运行 `npm run test` 进行连接测试

**相关资源**：
- [Midscene 文档 - 模型配置](https://midscenejs.com/model-provider.html)
- 代码位置：`midscene-example/connectivity-test/README.md`

## Q2: 连接测试失败怎么办？
**问题描述**：运行 `npm run test` 时出现错误，无法连接到 LLM API。

**答案**：
1. 检查 `.env` 文件是否存在且格式正确
2. 验证 API Key 是否正确
3. 检查网络连接
4. 查看错误信息，根据错误类型排查：
   - 如果是认证错误，检查 API Key
   - 如果是网络错误，检查网络连接
   - 如果是配置错误，检查环境变量配置

**相关资源**：
- [Midscene 文档 - 故障排查](https://midscenejs.com/zh)
- 代码位置：`midscene-example/connectivity-test/tests/`

## Q3: 支持哪些 LLM 模型？
**问题描述**：想了解 Midscene 支持哪些 LLM 模型，以及如何切换模型。

**答案**：
Midscene 支持多种 LLM 模型，包括：
- OpenAI（GPT-4o, GPT-4, GPT-3.5 等）
- Qwen（通义千问）
- 其他兼容 OpenAI API 的模型

切换模型的方法：
1. 修改 `.env` 文件中的配置
2. 参考 [Midscene 模型配置文档](https://midscenejs.com/choose-a-model)

**相关资源**：
- [Midscene 文档 - 选择模型](https://midscenejs.com/choose-a-model)
- 代码位置：`midscene/packages/core/src/ai-model/`

**标签**：#环境配置 #模型 #API
